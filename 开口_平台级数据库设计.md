# 开口平台级数据库设计

## 1. 设计目标

这份数据库设计不是只服务 V1 卡片训练，而是服务未来的表达训练平台。

它需要同时支持：
- 当前卡片联想训练
- 难度分层
- 会员体系
- 历史记录
- 管理后台
- AI 辩论
- 演讲朗读
- 语音分析
- TTS 语音播报

设计原则：
- 核心模型稳定
- 模式扩展不改主表
- 内容统一管理
- AI 反馈结构可扩展
- 后台统计可沉淀

## 2. 核心 ER 思路

```text
user
 ├── user_profile
 ├── user_membership
 ├── training_session
 │    ├── training_session_content
 │    ├── training_submission
 │    └── training_feedback
 ├── membership_order
 └── analytics_event

content_item
 ├── content_tag_relation
 ├── content_bundle_relation
 └── training_session_content

admin_user
 ├── admin_role
 └── operation_log
```

## 4. 内容域

## 4.1 `content_item`

用途：
- 统一内容中心，承载词语、辩题、演讲稿等

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | bigint | 主键 |
| content_uid | varchar(64) | 内容唯一 ID |
| content_type | varchar(32) | word / debate_topic / speech_script / speech_audio |
| title | varchar(255) | 标题或展示名 |
| body | text | 主内容 |
| extra_json | json | 扩展字段 |
| content_source_type | varchar(32) | preset / ai_generated / imported |
| difficulty_level | varchar(32) | beginner / intermediate / advanced |
| source | varchar(64) | 自建 / 导入来源 |
| status | varchar(32) | draft / published / offline |
| created_by | bigint | 创建人 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

说明：
- `word` 类型时，`title` 可直接存词语
- `speech_script` 类型时，`body` 存演讲稿全文
- `debate_topic` 类型时，`title` 存题目，`body` 存补充说明
- `extra_json` 用于存词性、领域、长度、语音链接、引用出处等扩展信息
- `content_source_type` 用于区分预置内容、AI 生成内容、导入内容

建议 `word` 类型在 `extra_json` 中至少支持：

```json
{
  "word_type": "noun",
  "domain_tag": "daily_life",
  "generation_prompt": "",
  "review_status": "approved"
}
```

## 10. V1 最低实现建议

虽然平台设计完整，但 V1 不必一次性把所有表都做满。

V1 最低必须落地：
- `user`
- `user_profile`
- `user_membership`
- `user_daily_quota`
- `content_item`
- `training_session`
- `training_session_content`
- `training_submission`
- `training_feedback`
- `card_round`
- `card_instance`
- `membership_order`
- `analytics_event`

V1 可先预留、后建：
- `content_generation_task`
- `speech_task`
- `daily_user_stats`
- `daily_training_stats`
- `daily_revenue_stats`
- 全套后台 RBAC 表

## 11. 最重要的架构判断

如果你只做卡片训练，旧表结构也能跑。

但如果你明确未来会做：
- AI 辩论
- 演讲训练
- 语音分析
- 语音播报
- AI 自动生成词库
- 按领域生成词语
- 管理后台

那现在就应该用这套平台级数据库设计，不然之后一定会返工。
