# 开口管理后台功能结构图

## 1. 后台目标

管理后台不是附属品，而是平台长期运营的基础设施。

它需要解决 4 件事：
- 看用户
- 看收入
- 管内容
- 看训练效果

## 2. 后台一级模块

建议后台包含 6 个一级模块：

1. 仪表盘
2. 用户管理
3. 订单与会员
4. 内容管理
5. 训练分析
6. 系统配置

## 4. 模块说明

## 4.1 仪表盘

### 目标
- 让管理者一进后台就知道平台运行情况

### 核心指标
- 总注册用户
- 今日新增用户
- DAU
- 今日训练次数
- 今日训练完成率
- 今日付费人数
- 今日收入

### 图表建议
- 最近 7 日用户增长曲线
- 最近 7 日训练次数趋势
- 最近 7 日收入趋势
- 各训练模式占比饼图

## 4.2 用户管理

### 用户列表
建议字段：
- 用户 ID
- 昵称
- 注册时间
- 最近活跃时间
- 是否会员
- 累计训练次数
- 平均得分

### 用户详情
建议展示：
- 基础信息
- 会员信息
- 训练概览
- 最近训练记录
- 支付记录

### 可执行操作
- 禁用用户
- 恢复用户
- 标记测试账号

## 4.3 订单与会员

### 订单列表
建议字段：
- 订单号
- 用户
- 商品类型
- 金额
- 支付状态
- 支付时间

### 统计看板
- 总支付金额
- 支付订单数
- 支付转化率
- 复购率

### 会员权益配置
建议支持：
- 价格配置
- 每日免费次数配置
- 会员可用功能配置

## 4.4 内容管理

### 词语管理
建议字段：
- 词语
- 类型
- 难度
- 标签
- 使用次数
- 状态

支持操作：
- 新增
- 编辑
- 上线
- 下线
- 批量导入

### 内容生成任务
建议支持：
- 按内容类型生成
- 按领域生成
- 按难度生成
- 预览生成结果
- 发布到正式内容池

建议字段：
- 任务 ID
- 内容类型
- 领域
- 难度
- 生成数量
- 任务状态
- 创建时间

### 辩题管理
建议字段：
- 辩题标题
- 正反方提示
- 难度
- 标签
- 使用次数
- 状态

### 演讲稿管理
建议字段：
- 标题
- 来源
- 难度
- 字数
- 状态

### 难度管理
建议至少支持：
- beginner
- intermediate
- advanced

## 4.5 训练分析

### 训练会话列表
建议字段：
- 会话 ID
- 用户 ID
- 模式
- 难度
- 开始时间
- 完成时间
- 时长
- 得分

### 训练详情
建议展示：
- 使用内容
- 用户提交
- AI 反馈
- 评分维度
- 语音分析结果

### 专项分析
- 卡片模式完成率
- 辩论模式使用率
- 演讲模式平均时长
- 各难度平均分
- 各模式用户留存差异

## 4.6 系统配置

### 训练模式配置
可配置：
- 是否开启
- 首页是否展示
- 默认难度
- 输入时长限制

### AI Prompt 配置
可配置：
- 卡片联想提示词
- 辩论模式提示词
- 演讲评分提示词

### 语音服务配置
可配置：
- ASR 服务商
- TTS 服务商
- 语音分析服务商

### 权限角色配置
建议角色：
- 超级管理员
- 运营
- 内容编辑
- 数据分析员

## 6. V1 最低后台范围

V1 不必一次做完整后台，建议先做最小运营后台：

### 必做
- 后台登录
- 仪表盘总览
- 用户列表
- 订单列表
- 词语内容管理

### 第二阶段再做
- 辩题管理
- 演讲稿管理
- 训练详情分析
- Prompt 配置
- RBAC 权限系统

## 7. 这个后台为什么重要

如果没有后台，后面会出现几个问题：
- 不知道用户到底练没练
- 不知道哪里付费转化最好
- 不知道哪些词、辩题、稿件效果好
- 不知道哪个训练模式最值得继续投资源

所以从长期看，后台不是锦上添花，而是平台扩展的基本盘。
