# 开口｜词语联想训练｜清晰版说明

## 先说结论

上一版不清楚，问题在我：

- 我把很多关键卡片省略成了占位框
- 我把动态逻辑讲在脑子里，没有完整翻译成状态稿
- 我把你要的“词语联想训练”画偏成了别的训练模式

这次我已经改成了“状态稿 + 规则稿”的表达方式。

## 产品规则

每轮训练：

1. 系统生成 `4×4` 矩阵，共 `16` 张卡
2. 词池固定为 `8 个名词 + 8 个形容词`
3. 用户在翻牌过程中自由选择 `2` 个词
4. 选满 `2` 词后，开始一次词语联想表达
5. 当 `16` 张卡全部翻开时，本轮结束
6. 免费用户最多 `3` 轮
7. 第 `4` 轮开始触发会员拦截
8. 下一轮词池必须与上一轮不重复

## Pencil 能不能做真正动态交互

不能完整做到。

Pencil 在这次环境里更适合做：

- 页面状态稿
- 信息结构稿
- 流程图
- 低保真界面

Pencil 不适合真实运行这些逻辑：

- 随机抽词
- 自动去重
- 真正翻牌
- 自动扣减免费次数
- 实时条件判断

所以这次正确的交付方式不是“可运行 Demo”，而是“把动态规则拆成清晰状态稿”。

## 三版差异

### A 版：单页主界面 + 底部抽屉表达

最推荐。

特点：

- 用户始终停留在同一主界面
- 先翻牌，再选 2 词
- 选满后底部抽屉展开
- 做完表达后回到同屏继续翻牌
- 最轻量，最适合首版

优点：

- 最不容易迷路
- 返回继续翻牌最顺
- 页面切换最少

风险：

- 抽屉打开后要注意不遮挡太多矩阵

### B 版：固定工作台

比 A 更“稳”，但更密。

特点：

- 上半区始终是翻牌矩阵
- 下半区始终是表达工作台
- 不靠抽屉或浮层
- 只是在工作台内部切换“待命/激活/完成”状态

优点：

- 用户永远知道表达应该在哪里做
- 最适合“边翻边想”

风险：

- 单屏信息密度更高
- 小程序首版会更重

### C 版：直入开局 + 任务条 + 表达 Sheet

最像小游戏。

特点：

- 用户进入即开局，不强调首页
- 先翻牌
- 选满 2 词后出现任务条
- 点击任务条拉起表达 Sheet
- 免费 3 轮结束后，像“继续闯关”一样触发会员门槛

优点：

- 启动阻力最低
- 更适合做连续挑战感
- 更容易形成产品记忆点

风险：

- 游戏感容易盖过训练感
- 如果表达任务太重，会显得不协调

## 我给你的默认建议

如果现在要做第一版产品验证，优先级建议：

1. A 版
2. B 版
3. C 版

原因：

- A 版最轻
- A 版最容易被用户理解
- A 版最适合先验证“4×4 词语联想矩阵”这个核心玩法
