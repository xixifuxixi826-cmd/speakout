- 个人主体阶段的快速上线验证
- 小红书 / 公众号 / 私域导流
- 外部下单 + 兑换码会员的轻量闭环

- 小程序前端继续保留：
  `/Users/lisa888/Documents/表达高手/代码原型/开口-miniapp-prototype`
- 后端继续共用：
  `/Users/lisa888/Documents/表达高手/backend`
- H5 前端单独新建：
  `/Users/lisa888/Documents/表达高手/代码原型/biaodagaoshou-h5`

这样做的好处：

- 不浪费现有小程序代码
- H5 可以按传播和转化来重新设计
- 后面如果切企业主体，小程序版还能继续推进

## 本地开发

本地开发时：

- H5 建议通过 `http://127.0.0.1:8088/` 访问
- 后端通过 `http://127.0.0.1:8765` 提供接口

本地启动方式：

```bash
python3 /Users/lisa888/Documents/表达高手/backend/server.py
python3 -m http.server 8088 --bind 127.0.0.1
```

## 正式部署

正式部署时：

- H5 和 API 建议挂在同一个域名下
- 前端会自动优先请求当前域名下的 `/api` 和 `/admin-api`
- 反向代理配置请看：

```text
/Users/lisa888/Documents/表达高手/deploy
```
