## Why

“开口”已经完成了产品方案、平台架构、数据库、API 和后台规划，但目前这些内容仍是独立文档，尚未形成可执行的变更定义。需要把 V1 首发范围固化为正式 change，确保后续开发围绕统一的能力边界推进，而不是退回到仅面向卡片玩法的一次性实现。

## What Changes

- 交付“开口”V1 首发能力，包括微信登录、会员权限、卡片联想训练、AI 反馈、历史记录和最小后台运营能力。
- 采用平台级数据模型，以 `training_session` 为核心主线，避免将实现写死为卡片训练专用系统。
- 引入统一内容中心，支持预置词库，并为未来 AI 动态生成词语和按领域生成内容预留来源与生成任务能力。
- 采用模块化 API 设计，按用户、会员、内容、训练、语音和后台域拆分接口。
- 交付 V1 最小后台，用于查看用户、订单、训练概览和管理词语内容。

## Capabilities

### New Capabilities
- `wechat-auth-membership`: 微信小程序登录、用户识别、每日额度和会员权限控制。
- `card-association-training`: 4x4 卡片联想训练流程、轮次管理、任意两词训练和历史记录沉淀。
- `ai-feedback`: 针对表达内容的固定 schema AI 反馈、会员分层展示和失败重试。
- `admin-operations`: V1 最小管理后台，支持仪表盘、用户列表、订单列表和词语管理。
- `content-generation`: 统一内容中心和内容来源策略，支持预置词库并为 AI 动态生成和按领域生成预留扩展能力。

## Impact

- 新增 OpenSpec 变更定义与对应 specs / design / tasks。
- 影响小程序前台、后台管理端、后端服务、数据库主表设计和 API 组织方式。
- 依赖微信登录与支付能力、AI 模型服务、ASR 服务，以及后续可扩展的内容生成能力。
