# 开口 V1 实施说明

## 1. 文档目的

本文件用于把 [PRD_开口_V1.md](/Users/lisa888/Documents/表达高手/PRD_开口_V1.md) 继续拆解为可执行层内容，便于设计、前端、后端和 AI 接口联调直接使用。

适用对象：
- 产品经理
- UI/UX 设计师
- 小程序前端开发
- 后端开发
- AI 接口开发

## 2. 页面清单

V1 建议共 7 个页面 / 弹层：

1. 首页
2. 卡片训练页
3. 构思倒计时页
4. 表达输入页
5. AI 反馈结果页
6. 历史记录列表页
7. 历史记录详情页

补充弹层：
- 本轮完成弹窗
- 免费次数用尽弹窗
- 开通会员弹窗
- AI 生成失败弹窗

## 3. 页面原型文案

## 3.1 首页

### 页面目标
让用户快速进入训练，并清楚知道自己今天还能练几次。

### 页面结构
- 顶部标题：`开口`
- 副标题：`把两个词讲出一个观点`
- 训练信息区
- 主按钮区
- 辅助入口区

### 页面文案建议

主标题：
`开口`

副标题：
`随手抽两个词，讲出你的联系`

训练信息区：
- `今日还可训练 2 次`
- `最近一次得分：7.8`
- `你已经连续训练 3 天`

主按钮：
- `开始训练`

辅助入口：
- `历史记录`
- `开通会员`

### 空状态
- 新用户首次进入：
  - `今天先开口一次，别想太多`
  - `没有标准答案，重点是把你的想法讲出来`

### 次数用尽状态
- `今日免费次数已用完`
- `开通会员，继续无限训练`

按钮：
- `开通会员继续`

## 3.2 卡片训练页

### 页面目标
完成翻卡、选词和进入表达前准备。

### 页面结构
- 顶部进度区
- 4 x 4 卡片矩阵
- 当前选中词语区
- 底部操作区

### 页面文案建议

顶部：
- `本轮进度 6 / 16`
- `任选两个词，讲出一个能成立的观点`

提示文案：
- `不需要考虑词性搭不搭`
- `越远的两个词，越能练表达`

选中区：
- `已选择：聪明 + 愚蠢`

按钮状态：
- 未选满 2 张时：`请选择 2 张卡片`
- 选满后：`开始表达`

### 交互状态
- 未翻开：仅展示卡背
- 已翻开：展示词语
- 已选中：高亮边框
- 已使用：灰掉，不可点击

### 错误提示
- 已选 2 张后继续点击其他卡片：
  - `每次只能选择 2 张卡`

## 3.3 构思倒计时页

### 页面目标
给用户一个短暂缓冲，快速组织观点。

### 页面文案建议
- 标题：`准备开口`
- 倒计时主文案：`30`
- 提示文案：
  - `请围绕这两个词，建立一个尽可能成立的观点`
  - `没有标准答案，重点是讲清楚你的联系`

按钮：
- `跳过倒计时，直接开始`

## 3.4 表达输入页

### 页面目标
让用户完成一次语音或文本表达。

### 页面结构
- 当前词语展示
- 输入方式切换
- 语音输入区
- 文本输入区
- 字数 / 时间提示
- 提交按钮

### 页面文案建议

当前词语：
- `本次词语：聪明 + 愚蠢`

语音模式：
- `按住开始说`
- `松开结束`
- `系统会实时转成文字`

文本模式：
- 输入框 placeholder：
  - `把你想到的观点写下来`

限制提示：
- `建议 30 秒到 2 分钟`
- `最多 300 字`

提交按钮：
- `提交分析`

### 异常文案
- ASR 失败：
  - `语音识别失败，你也可以直接输入文字`
- 超字数：
  - `已超过 300 字，请精简后提交`
- 空内容：
  - `请先说点什么，再提交`

## 3.5 AI 反馈结果页

### 页面目标
帮助用户立刻理解本次表达表现，并愿意继续下一次训练。

### 页面结构
- 总评分区
- 四维评分区
- 具体问题点评区
- 优化建议区
- 改写对比区
- 底部操作区

### 页面文案建议

总评分：
- `本次得分 7.8`

评分解释：
- `你已经成功把两个词联系起来了，但核心观点出现得偏晚`

四维评分：
- `结构清晰度：7`
- `逻辑合理性：8`
- `表达具体性：6`
- `语言流畅度：8`

问题点评标题：
- `这次最值得改的一点`

优化建议标题：
- `下次可以这样说`

改写对比标题：
- `如果表达得更顺一点`

底部按钮：
- `继续训练`
- `保存并返回首页`

### 免费版文案
- `查看完整建议与改写版本，开通会员即可解锁`

按钮：
- `开通会员查看完整反馈`

## 3.6 历史记录列表页

### 页面目标
帮助会员快速回看自己的训练表现。

### 页面文案建议

页面标题：
- `历史记录`

列表项信息：
- `04-18 19:40`
- `得分 8.2`
- `标签：观点清楚`

空状态：
- `还没有历史记录`
- `先完成一次训练，再回来看看`

## 3.7 历史记录详情页

### 页面目标
完整回看单次训练内容与 AI 反馈。

### 页面文案建议
- `词语组合：聪明 + 愚蠢`
- `我的原话`
- `AI 评分`
- `AI 建议`
- `AI 改写`

底部按钮：
- `再练一次`

## 4. 功能清单

## 4.1 账号与权限

### 必做
- 微信小程序登录
- 获取并绑定 `openid`
- 识别是否为会员
- 基于会员状态控制次数和内容权限

### 规则
- 免费用户每日 3 次
- 会员用户无限次
- 历史记录仅会员可见
- 完整 AI 反馈仅会员可见

## 4.2 卡片训练

### 必做
- 初始化 16 张卡
- 按规则抽取 8 个名词和 8 个形容词
- 同轮去重
- 用户当天词语去重
- 记录卡片状态流转
- 允许任意 2 张卡进入训练

### 状态机

卡片状态：
- `0 未翻开`
- `1 已翻开`
- `2 已使用`

训练状态：
- `idle`
- `selecting`
- `thinking`
- `speaking`
- `analyzing`
- `completed`

## 4.3 语音与文本输入

### 必做
- 微信端语音采集
- ASR 实时或准实时转文字
- 文本输入兜底
- 300 字长度限制
- 2 分钟时长限制

### 不做
- 音频存储
- 音频回放
- 语音质量评分

## 4.4 AI 反馈

### 必做
- 提交词语 1、词语 2、用户文本
- 返回固定 JSON 结构
- 免费版和会员版输出控制
- 存储评分、建议、改写结果

### AI 判断原则
- 重点看是否建立联系、讲清观点
- 不以词性逻辑否定表达
- 点评必须具体

## 4.5 历史记录

### 必做
- 训练完成后写入记录
- 支持列表查询
- 支持详情查询
- 支持标签展示

## 5. 接口草案

以下路径为建议形式，实际可按团队规范调整。

## 5.1 登录与用户信息

### `POST /api/auth/wechat/login`

用途：
- 使用微信登录 code 换取用户登录态

请求体：

```json
{
  "code": "wx_login_code"
}
```

响应体：

```json
{
  "token": "user_token",
  "user": {
    "user_id": "openid_xxx",
    "is_vip": false,
    "daily_remaining_count": 3
  }
}
```

### `GET /api/user/profile`

用途：
- 获取用户基本信息和当天训练状态

响应体：

```json
{
  "user_id": "openid_xxx",
  "is_vip": false,
  "daily_remaining_count": 2,
  "last_score": 7.8,
  "streak_days": 3
}
```

## 5.2 轮次与卡片

### `POST /api/rounds/current`

用途：
- 获取当前未完成轮次
- 若不存在则创建新轮次

响应体：

```json
{
  "round_id": "round_001",
  "progress_used_count": 6,
  "total_count": 16,
  "cards": [
    {
      "card_id": "card_001",
      "word": "",
      "type": "noun",
      "status": 0
    }
  ]
}
```

说明：
- 未翻开卡片可不返回具体词语，避免前端提前泄露内容

### `POST /api/cards/flip`

用途：
- 翻开单张卡片

请求体：

```json
{
  "card_id": "card_001"
}
```

响应体：

```json
{
  "card_id": "card_001",
  "word": "聪明",
  "type": "adj",
  "status": 1
}
```

### `POST /api/rounds/next`

用途：
- 当前轮完成后，创建下一轮

请求体：

```json
{
  "finished_round_id": "round_001"
}
```

响应体：

```json
{
  "round_id": "round_002",
  "cards": []
}
```

## 5.3 训练提交

### `POST /api/trainings/analyze`

用途：
- 提交一次训练并触发 AI 分析

请求体：

```json
{
  "round_id": "round_001",
  "selected_card_ids": ["card_001", "card_008"],
  "word1": "聪明",
  "word2": "愚蠢",
  "user_text": "有时候一个看起来愚蠢的人反而更聪明，因为他知道什么时候不去逞强。"
}
```

响应体：

```json
{
  "record_id": "record_001",
  "score": {
    "overall": 7.8,
    "structure": 7,
    "logic": 8,
    "specificity": 6,
    "fluency": 8
  },
  "feedback": {
    "issue": "你的核心判断是明确的，但缺少一个更具体的例子支撑，所以说服力还可以再加强。",
    "suggestions": [
      "开头先直接说出你的观点。",
      "补一个具体情境来证明你的判断。"
    ],
    "rewrite_text": "有时候真正聪明的人，反而不会急着证明自己聪明。一个看起来有些愚蠢的人，可能只是选择了克制和不逞强，这恰恰是一种更成熟的聪明。"
  },
  "privilege": {
    "history_available": false,
    "full_feedback_available": false
  }
}
```

### `POST /api/trainings/retry-ai`

用途：
- AI 分析失败时重试

请求体：

```json
{
  "record_id": "record_001"
}
```

## 6. AI 返回 JSON 规范

建议后端统一为固定结构，避免前端适配复杂度升高。

```json
{
  "overall_score": 0,
  "dimension_scores": {
    "structure_clarity": 0,
    "logic_connection": 0,
    "specificity": 0,
    "fluency": 0
  },
  "issue_comment": "",
  "suggestions": ["", ""],
  "rewrite_text": "",
  "tag": ""
}
```

字段要求：
- `overall_score`：0 到 10，可保留 1 位小数
- `dimension_scores`：四项 0 到 10 分
- `issue_comment`：至少 1 个具体问题
- `suggestions`：1 到 3 条
- `rewrite_text`：必填
- `tag`：用于历史记录列表，例如“结构弱”“不够具体”

## 7. 开发优先级

## P0
- 微信登录
- 首页
- 卡片训练主流程
- 语音转文字
- AI 分析
- 结果页
- 免费次数限制

## P1
- 历史记录
- 会员支付
- 完整反馈解锁
- 当天词语去重优化

## P2
- 连续训练天数
- 标签体系优化
- 更丰富的词库运营工具

## 8. 风险与建议

## 8.1 当前最大风险
- 微信端 ASR 体验是否稳定
- AI 返回时延是否影响完成率
- 词库量不足会导致去重策略过早失效

## 8.2 建议
- 词库初版至少准备 300 到 500 个日常词
- 先跑通文字输入链路，再接语音输入
- AI 输出必须走固定 JSON 模板
- 免费版和会员版的差异尽量控制在“展示权限”，而不是两套完全不同逻辑
